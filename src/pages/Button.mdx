# Button

The `Button` component enables users to trigger actions with a single click. It supports multiple color variants for different semantic purposes and can include icons.

---

## Props

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `children` | `string` | Yes | - | Button text label |
| `color` | `InteractableColor` | Yes | - | `"positive"`, `"primary"`, `"negative"`, `"dark"`, `"light"` |
| `onClick` | `function` | Yes | - | Click handler |
| `bare` | `boolean` | No | `false` | Removes background for a lighter appearance |
| `border` | `boolean` | No | `false` | Adds a border matching the text color |
| `disabled` | `boolean` | No | `false` | Disables interaction |
| `icon` | `IconType` | No | - | Icon to display alongside text |
| `iconPosition` | `"left"` \| `"right"` | No | `"left"` | Icon placement relative to text |
| `iconSize` | `Size` | No | `"medium"` | Icon size |
| `size` | `Size` | No | `"medium"` | Text size |
| `padding` | `Padding` | No | `"medium"` | Internal padding |
| `bold` | `boolean` | No | `false` | Bold text |

---

## Color variants

Each color variant communicates a different intent to users:

- **Positive**: Confirmations, submissions, primary actions
- **Primary**: Secondary or neutral actions
- **Negative**: Destructive actions like delete or remove
- **Dark/Light**: Theme-specific variants for special contexts

  <Button color="positive" onClick={() => {}}>Positive</Button>
  <Button color="primary" onClick={() => {}}>Primary</Button>
  <Button color="negative" onClick={() => {}}>Negative</Button>
  <Button color="dark" onClick={() => {}}>Dark</Button>
  <Button color="light" onClick={() => {}}>Light</Button>
</Row>


## Bare variant

The `bare` prop removes the background color, creating a lighter appearance that integrates well within complex layouts or when you need a less prominent action.

<Row gap="large" align="center" padding="large" justify="center">
  <Button bare color="positive" onClick={() => {}}>Positive</Button>
  <Button bare color="primary" onClick={() => {}}>Primary</Button>
  <Button bare color="negative" onClick={() => {}}>Negative</Button>
  <Button bare color="dark" onClick={() => {}}>Dark</Button>
  <Button bare color="light" onClick={() => {}}>Light</Button>
</Row>

```js
<Button bare color="positive" onClick={() => {}}>Positive</Button>
<Button bare color="primary" onClick={() => {}}>Primary</Button>
<Button bare color="negative" onClick={() => {}}>Negative</Button>
```

## With icon

Buttons can include icons to reinforce their purpose. Use `icon` to specify the icon and `iconPosition` to place it on the left or right.

<Row gap="large" align="center" padding="large" justify="center">
  <Button color="positive" icon="fa-check" onClick={() => {}}>Confirm</Button>
  <Button color="negative" icon="fa-trash" onClick={() => {}}>Delete</Button>
  <Button color="primary" icon="fa-arrow-right" iconPosition="right" onClick={() => {}}>Next</Button>
</Row>

```js
<Button color="positive" icon="fa-check" onClick={() => {}}>
  Confirm
</Button>

<Button color="negative" icon="fa-trash" onClick={() => {}}>
  Delete
</Button>

<Button color="primary" icon="fa-arrow-right" iconPosition="right" onClick={() => {}}>
  Next
</Button>
```

## Disabled state

Disabled buttons prevent interaction and display a muted appearance. The component uses `aria-disabled` for accessibility.

<Row gap="large" align="center" padding="large" justify="center">
  <Button color="positive" disabled onClick={() => {}}>Disabled</Button>
  <Button bare color="positive" disabled onClick={() => {}}>Disabled bare</Button>
  <Button border bare color="positive" disabled onClick={() => {}}>Disabled outlined</Button>
</Row>

```js
<Button color="positive" disabled onClick={() => {}}>
  Disabled
</Button>
```

</Row>

```js
<Button color="positive" size="small" onClick={() => {}}>Small</Button>
<Button color="positive" size="medium" onClick={() => {}}>Medium</Button>

```

---

## Accessibility

- Buttons use semantic `<button>` elements
- Disabled state is communicated via `aria-disabled` attribute
- Focus states are clearly visible with outline styling
- Click handlers are ignored when disabled, preventing accidental actions
